# Task: Investigate Design System / Visual Consistency

## Objective
Research the overall design system - CSS variables, shared styles, component consistency. Understand how cohesive or fragmented it is. DO NOT MAKE CHANGES - research only.

## Questions to Answer
1. Where are CSS variables defined? How many? How organized?
2. Is there a design token system or ad-hoc?
3. Are components using consistent spacing/colors/typography?
4. What's the color palette? Is it documented/systematic?
5. Are there multiple button styles? Input styles? Panel styles?
6. What would "consistent component styling" require?
7. Is there a base/reset CSS? Global styles?
8. How fragmented vs unified is the current approach?

## Deliverable
Write a report to `./reports/investigate-design-system.md` with:
- Inventory of CSS files and their purposes
- CSS variable audit (what exists, what's missing)
- Consistency assessment (where things diverge)
- Color palette summary
- Typography summary
- Component style inventory (buttons, inputs, panels, etc.)
- Recommendations for improving cohesion
- Estimated effort to unify

## Rules
- READ ONLY - no edits
- Create an inventory/audit
- Be honest about the current state
